# 功能测试指南

## 测试前准备

1. **重启服务器**：
   ```bash
   cd /Users/mac/Desktop/work/co-editor
   node server.js
   ```

2. **清除浏览器缓存**：
   - 打开开发者工具（F12）
   - 右键刷新 → "清空缓存并硬性重新加载"

---

## 测试 1：文件下载（修复重复下载）

### 步骤

1. **创建或打开一个文档**

2. **拖拽文件到编辑器**：
   - 可以是图片、文档等任意文件
   - 确保文件出现在文件列表中

3. **刷新页面多次**：
   - 第1次刷新
   - 第2次刷新
   - 第3次刷新

4. **点击"下载"按钮**

5. **检查下载文件夹**：
   - Windows: `C:\Users<用户名>\Downloads\`
   - macOS: `/Users/<用户名>/Downloads/`
   - Linux: `/home/<用户名>/Downloads/`

### 预期结果
- ✅ 只下载 **1份** 文件
- ❌ 不应该有重复文件

### 如果失败了
- 检查浏览器控制台是否有错误
- 查看下载文件夹，统计文件数量
- 记录文件名和大小

---

## 测试 2：分享功能

### 步骤

1. **进入任意文档**（可以是新创建的也可以是已有的）

2. **点击"🔗 分享"按钮**：
   - 位置：页面顶部，"文档列表"按钮右侧
   - 应该弹出一个模态框

3. **检查模态框内容**：
   - [ ] 显示文档链接
   - [ ] 显示"复制"按钮
   - [ ] 显示二维码生成区域
   - [ ] 显示提示信息

4. **复制链接测试**：
   - 点击"复制"按钮
   - 应该显示"✓ 已复制到剪贴板"
   - 粘贴到记事本验证

5. **二维码测试**：
   - 等待二维码生成完成
   - 使用手机摄像头扫描二维码
   - 应该能打开文档

6. **关闭模态框**：
   - 点击"关闭"按钮
   - 或点击模态框外部区域
   - 模态框应该关闭

### 预期结果
- ✅ 分享按钮正常显示
- ✅ 模态框正确弹出
- ✅ 链接复制成功
- ✅ 二维码生成成功
- ✅ 扫描二维码能访问文档

### 如果失败了
- 记录失败的具体步骤
- 检查浏览器控制台是否有错误
- 截图发送给开发者

---

## 测试 3：分享功能在不同场景

### 场景 A：公开文档
1. 创建一个不带密码的文档
2. 分享链接给他人
3. 对方点击链接
4. **应该能直接进入，无需密码**

### 场景 B：私密文档
1. 创建一个带密码的文档（如密码 "123456"）
2. 分享链接给他人
3. 对方点击链接
4. **应该提示输入密码**

### 场景 C：复制到不同应用
1. 复制链接
2. 粘贴到：
   - QQ/微信聊天窗口
   - 邮件
   - 记事本
   - 浏览器地址栏

5. 每次粘贴都应该能正常使用

---

## 测试 4：集成测试（综合）

### 测试场景：多人协作编辑 + 文件共享

**用户 A 操作**：
1. 创建一个文档
2. 写入一些内容
3. 分享链接给用户 B
4. 拖拽一个文件到编辑器

**用户 B 操作**：
1. 接收用户 A 的分享链接
2. 进入文档
3. 应该看到用户 A 的内容
4. 编辑添加更多内容
5. 用户 A 应该能看到实时更新
6. 下载用户 A 分享的文件
7. **应该只下载1份文件**

### 预期结果
- ✅ 多人实时编辑正常
- ✅ 文件共享正常
- ✅ 下载功能正常（不重复）
